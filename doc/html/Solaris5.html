5. Known bugs, and work arounds with Solaris

  1. The Solaris 2.1 for x86 OpenWindows filemgr does not work against a X11R5 Xserver, it probably will also not work against a X11R6 Xserver. Attempting to 'Drag and Drop' a file causes the filemgr to abort with an 'X error'

    Solaris x86 2.4 does not have this problem.

    There is no known work around.

  2. The SPARC port is still quite new, so instability is to be expected (and reported!). It might even have broken some aspects of the x86 port.
  3. On SPARC, xload should be updated to deal with both 32-bit and 64-bit SunOS kernels. It currently is compiled as a 32-bit binary and cannot handle a 64-bit kernel.
